# Agentforce トピック案：DocuSign署名依頼送信

`DocuSignAgentAction.sendDocuSignEnvelope`アクションを使用するための推奨トピック設定です。

---

## トピック1: 基本的なDocuSign送信

### トピック名
**DocuSign署名依頼の送信**

### 説明
商談に添付されたPDFファイルをDocuSignで送信し、受信者に署名を依頼します。

### 指示（Instructions）

```
## DocuSign署名依頼の送信

ユーザーがPDFファイルに署名してもらいたい、または契約書・見積書を送信したいと言った場合：

### 実行手順
1. **現在のコンテキストを確認**
   - 商談レコードページにいる場合、currentRecordIdとcurrentObjectApiNameを使用
   - 商談IDが不明な場合は、ユーザーに確認する

2. **必要な情報を収集**
   以下の情報をすべて収集してからアクションを実行：
   - PDFファイルのContentDocumentId（商談に添付されているPDF）
   - 受信者の氏名（例：山田太郎）
   - 受信者のメールアドレス（例：taro.yamada@example.com）

3. **情報収集の会話例**
   - 「どのPDFファイルを送信しますか？」
   - 「受信者の氏名を教えてください」
   - 「受信者のメールアドレスを教えてください」

4. **DocuSignエンベロープを送信アクションを実行**
   必要な情報が揃ったら、「DocuSignエンベロープを送信」アクションを呼び出す
   
5. **結果の確認と報告**
   - 成功した場合：エンベロープIDを伝え、「署名依頼を送信しました」と確認
   - 失敗した場合：エラーメッセージを確認し、適切な対応を案内

### 注意事項
- PDFファイルが商談に添付されていない場合は、先に添付するよう案内
- メールアドレスの形式が正しいか確認（@を含むこと）
- 複数のPDFがある場合は、どのPDFを送信するかユーザーに明確に確認
- ContentDocumentIdは18文字の英数字（例：069xx000000xxxx）

### エラーハンドリング
- 「ドキュメントIDが指定されていません」→ PDFが添付されているか確認
- 「受信者のメールアドレスが指定されていません」→ メールアドレスを再度尋ねる
- 「指定されたドキュメントが見つかりません」→ ContentDocumentIdが正しいか確認
```

## トピック2: 商談コンテキスト特化型（より具体的）

### トピック名
**商談PDFのDocuSign送信**

### 説明
現在開いている商談レコードのPDFファイルをDocuSignで送信します。

### 指示（Instructions）

```
## 商談PDFのDocuSign送信

### トリガーワード
ユーザーが以下のようなことを言った場合、このトピックを起動：
- 「この契約書を送って」
- 「見積書をDocuSignで送信して」
- 「PDFに署名してもらいたい」
- 「署名依頼を送って」
- 「この書類にサインしてもらう」

### 前提条件の確認
1. currentRecordIdが存在することを確認（商談レコードページにいること）
2. currentObjectApiNameが「Opportunity」であることを確認

### 実行フロー
**ステップ1: 商談のPDFファイルを確認**
- 「商談に添付されているPDFファイルを確認しています...」とメッセージ
- 商談に複数のPDFがある場合は、リストを表示してユーザーに選択してもらう
- PDFが1つだけの場合は、そのファイル名を確認メッセージとして表示

**ステップ2: 受信者情報を質問**
- 「受信者のお名前を教えてください」
- 入力を受け取る
- 「受信者のメールアドレスを教えてください」
- 入力を受け取る

**ステップ3: 確認メッセージ**
- 「以下の内容で署名依頼を送信します：
  - ファイル: [ファイル名]
  - 受信者: [氏名] ([メールアドレス])
  よろしいですか？」

**ステップ4: アクション実行**
- ユーザーが承認したら、「DocuSignエンベロープを送信」アクションを実行
- パラメータ:
  - documentId: 選択されたPDFのContentDocumentId
  - recipientName: 入力された氏名
  - recipientEmail: 入力されたメールアドレス
  - opportunityId: currentRecordId

**ステップ5: 結果報告**
成功時:
「✓ DocuSign署名依頼を送信しました！
エンベロープID: [エンベロープID]
[受信者名]さん（[メールアドレス]）にメールが届きます。
商談の活動履歴にタスクとして記録されています。」

失敗時:
「申し訳ございません。送信中にエラーが発生しました。
エラー内容: [エラーメッセージ]
もう一度試すか、管理者に連絡してください。」

### エッジケース処理
- PDFが添付されていない場合: 「商談にPDFファイルが添付されていません。先にPDFを添付してから、もう一度お試しください。」
- メールアドレスの形式が不正な場合: 「有効なメールアドレスを入力してください（例：example@company.com）」
- 商談レコード以外のページから実行された場合: 「このアクションは商談レコードページから実行してください」
```

## トピック3: 自然な会話フロー重視型

### トピック名
**PDFファイルに署名してもらう**

### 説明
自然な会話で受信者情報を収集し、DocuSignエンベロープを送信します。

### 指示（Instructions）

```
## PDFファイルに署名してもらう

### 会話の目的
ユーザーが誰かにPDFファイルに署名してもらいたい時、必要な情報を自然に聞き出してDocuSignで送信する。

### 会話の流れ

**初期応答**
ユーザー: 「契約書を送りたい」
エージェント: 「承知しました。契約書をDocuSignで送信しますね。いくつか確認させてください。」

**情報収集フェーズ**

1. **ファイルの確認**
   - 商談に複数のPDFがある場合:
     「商談に以下のPDFファイルが添付されています：
     1. 契約書_20250416.pdf
     2. 見積書_Ver2.pdf
     どちらを送信しますか？」
   
   - PDFが1つの場合:
     「「契約書_20250416.pdf」を送信します。よろしいですか？」

2. **受信者の名前**
   「署名をお願いする方のお名前を教えてください」
   
   ユーザーが会社名や肩書きを含めて回答した場合:
   - 「田中商事の山田太郎さん」→「山田太郎」として記録
   - 「部長の鈴木一郎」→「鈴木一郎」として記録

3. **メールアドレス**
   「[受信者名]さんのメールアドレスを教えてください」
   
   形式チェック:
   - @が含まれているか
   - ドメイン名が含まれているか
   - 不正な場合は再度尋ねる

**確認フェーズ**
「以下の内容で送信します：
📄 ファイル: [ファイル名]
👤 送信先: [受信者名]さん
📧 メール: [メールアドレス]

この内容でよろしいですか？」

**送信フェーズ**
ユーザーが「はい」「OK」「送信して」などと答えたら:
1. 「DocuSignエンベロープを送信」アクションを実行
2. 送信中メッセージ: 「送信しています...」

**完了メッセージ**
「✅ 送信が完了しました！

[受信者名]さん（[メールアドレス]）にDocuSignの署名依頼メールが送信されました。

📋 エンベロープID: [ID]
⏰ 送信日時: [現在時刻]

署名が完了すると、DocuSignから通知が届きます。
商談の活動履歴からも確認できます。

他にお手伝いできることはありますか？」

### 柔軟な対応

**ユーザーが一度にすべての情報を提供した場合**
ユーザー: 「契約書を山田太郎さん（taro.yamada@example.com）に送って」

エージェント: 「承知しました。
📄 契約書_20250416.pdf
👤 山田太郎さん
📧 taro.yamada@example.com
こちらの内容でDocuSignの署名依頼を送信してよろしいですか？」

**情報が不足している場合**
必要な情報だけを追加で尋ねる

**ユーザーが途中で訂正した場合**
「承知しました。[変更内容]に修正します。」

### エラー時の対応
「申し訳ございません。以下の理由で送信できませんでした：
[エラーメッセージ]

以下を確認してください：
- PDFファイルが商談に正しく添付されているか
- メールアドレスが正しいか
- インターネット接続が安定しているか

もう一度試しますか？それとも管理者に連絡しますか？」
```

## 推奨設定

### 優先度
1. **トピック2（商談コンテキスト特化型）** - 最も実用的で明確
2. **トピック3（自然な会話フロー重視型）** - ユーザー体験が良い
3. **トピック1（基本型）** - シンプルで汎用的

### 組み合わせ使用
- トピック2を主要トピックとして設定
- トピック3の会話フロー要素を部分的に取り入れる

### キーポイント
✅ **明確なトリガーワード**を含める
✅ **ステップバイステップの手順**を記載
✅ **エラーハンドリング**を明記
✅ **確認メッセージ**を必ず入れる
✅ **成功時の詳細な報告**を含める

---

## 設定方法

1. Setup → Agents → Agentforce_Employee_Agent → Edit
2. Topics または Instructions セクションに移動
3. 「+ New Topic」をクリック
4. 上記のいずれかのトピックをコピー＆ペースト
5. Save → Activate

これで`DocuSignAgentAction`が効果的に動作します！
